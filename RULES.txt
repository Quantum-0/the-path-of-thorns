Здесь будут описаны правила ведения репозитория

Ветки:
- master - защищена, для стабильных релизов
- develop - общая для разработки, защищена для пуша, разрешён мерж в неё
- art - для художников (, аниматоров и т.д.), при необходимости мержится в develop прогерами(!)
- ветки для разработки создаём себе сами, по окончанию разработки конкретной фичи - мерж в develop

Коммиты:
- Коммит должен иметь короткое поясняющее название о том, что он содержит
- Коммит содержит конкретные изменения, затрагивающие только одну какую-то фичу. Коммит вида "исправил движение игрока и добавил врага" должны быть разбит на два отдельных коммита
- Некомпилящиеся, косячные изменения коммитятся с префиксом "WiP:"
- При мерже в develop - ревью изменений (как минимум по коду) делает другой программист. Одна голова хорошо, а вторая увидит косяки первой ;) плюс это поможет участникам быть ознакомленными с кодом друг друга

Задачи:
- Задачи предлагаю вести в трелло
- Доски для прогеров, хуёжников, композиторов и т.п.
- Колонки - TODO, IN PROGRESS, TESTING, DONE
- TODO - добавляем сюда карточки, описываем в них что надо сделать
- Из TODO переносим в IN PROGRESS и помечаем себя как исполнителя задачи, эту задачу больше никто не трогает (за исключением необходимости помочь)
- По завершению задачи переносим в TESTING
- Если внесённое изменение протестировано и всё успешно работает - перекидываем в DONE

Баги:
- В Issues наверно лучше всего заносить?

Как работают хуёжники:
- Pull ветки Art
- Добавляем файлы, Commit
- Push ветки Art

Как работают программисты:
- Ответвляемся из develop в ветку фичи, которую пилим, например - добавляем нового врага
- Вносим изменения
- После того как сделали - проверяем, не поломалось ли из-за наших изменений что-то другое
- мержим develop в нашу ветку (!), если та изменялась. Разрешаем конфликты
- мержим нашу ветку в develop

Структура проекта:
- тут напишем какие папки есть в проекте, какие в них файлы, что они делают
- например:
- Scripts - папка со скриптами, содержит .cs файлы
- Prefabs - префабы для проекта
- Images/Sprites - спрайты используемые в игре
- Images/Background - фоновые изображения
- Images/UI - кнопочки, окошки, прочие элементы интерфейса

Альтернативный вариант организации работы для художников - отдельный репозиторий, подключённый к проекту как сабмодуль
Тогда исключаем ветку art, добавляются/изменяются арты в отдельном репозитории в одной единственной ветке, для обновления тут просто обновляем сабмодуль
